# “吃点好的” — 智能治愈系饮食记录应用设计文档 (v2.0 优化版)

本项目是一款面向日常生活记录的智能饮食 Web 应用，以“吃点好的”为核心主题。将日常饮食记录转化为“可回忆、可收藏的生活回忆录”。

---

## 1. 核心视觉与页面规划

### 1.1 视觉风格 (UI/UX)
* **配色调性**：采用柔和温润的粉彩马卡龙色调。
  * 背景色：暖奶油白（#FFFDF6）辅以可爱的网格线背景，模拟手账本的网格纸纹理。
  * 卡片及辅助色：粉色（#FFB6C1）、薄荷绿（#A8E6CF）、香蕉黄（#FFD3B6）。
  * 交互体验：大圆角卡片（`border-radius: 24px`），按钮和过渡带有弹性 bouncy 动画。
* **Logo 设计**：
  * 顶部 Logo 采用手绘圆体排版，左侧配有由纯 CSS 和 SVG 绘制的**热气腾腾小面碗/可爱笑脸饭团**图形，文字下方带有波浪虚线装饰。
* **可爱字体引入**：
  * 全局引入 Google Fonts 的中文字体 **"ZCOOL KuaiLe" (站酷快乐体)**，英文及数字搭配 **"Fredoka"**，使整个界面具有萌系治愈的手账笔触风格。
* **不规则轮廓食物贴纸**：
  * 上传的食物照片经本地 WASM 自动去除背景后，利用 **Canvas 轮廓提取算法**（检测非透明像素边缘）获取食物本身的真实轮廓路径，在其轮廓周围绘制 4px 白色虚线描边，实现贴合食物边界的实体贴纸质感（而不是矩形框描边）。

### 1.2 四大核心功能板块
1. **【食光时间轴 - Today】(首页)**
   * **结绳记事概念**：采用一条连贯的垂直时间轴。每一个日期是一个圆形的“绳结节点”（例如“7月2日”）。
   * 日期节点下方依次展开该天所有的饮食卡片。
   * 用户可以直接向下滑动浏览过去的所有历史记录，彻底取消顶部的时间筛选翻页器。
2. **【月视图】**
   * 以传统的日历网格展示整月。
   * 每个日期格中会平铺展示当天所有饮食记录的食物贴纸缩略图，形成一面绚丽的“当月食物贴纸墙”。
   * 点击日期格直接平滑滚动定位到首页该日期的位置。
3. **【回忆录】(月报模式)**
   * 自动生成一份精美的长图卡片，展示当月的数据回忆：
     * **AI 温暖总结**：调用 AI（Gemini）根据用户当月的饮食历史、聚餐频次和心情，生成一段温暖治愈的月度总结（例如：“这个月你尝试了许多新的味道，也和重要的人度过了温暖的饭局...”）。
     * **趣味统计指标**：
       * 🍽️ 累计用餐次数
       * 🆕 尝鲜新食物（首次记录的食物数）
       * 🥤 饮品次数（统计名字中包含奶茶、咖啡、可乐、饮品等的记录）
       * 👥 朋友聚餐次数（统计有“和谁吃”的记录数）
       * 💖 最偏爱与最常吃（基于评分与词频统计）
       * 🌙 深夜食光（21:00 - 4:00 最晚用餐记录）
       * 📍 常去地点（“在哪里吃”的地点频次分析）
     * 提供一键“导出为图片”功能。
4. **【收藏推荐页】**
   * 展示所有被用户标记为“收藏”的饮食回忆卡片，形成个人专属的“美食推荐清单”和“生活高光时刻”。

### 1.3 Icon 与导航改版
* 彻底弃用 Emoji 风格。导航栏与卡片内全部替换为简约流畅的 **Lucide 线性图标**（今日-`Calendar`，月视图-`Grid`，回忆录-`BarChart3`，收藏-`Heart`）。
* **底部导航栏只显示简约 Icon**，点击时带有缩放和颜色填充过渡的微动效，不包含多余文字。

## 2. 核心交互流程

### 2.1 “吃点好的” 记录新增 (RecordModal) 界面精修
* **拍立得相纸上传区**：上传区域重新设计为精致的白色拍立得相框样式，带底边白色手写字留白区。
* **手账式表单排版**：
  * 去除臃肿厚重的输入框边框，改为**简洁的下划线虚线**（`border-bottom: 2px dashed #E8D3C5`）。
  * 选项卡（餐时选择）采用药丸状组合，按钮采用极简线框，选中时显示背景色填充。
  * 评分采用粉色手绘感心形图标，带有 Hover 缩放动画。
  * 字段整齐地划分为三个层级：“什么食物？”、“和谁在哪里吃？”、“美味心得与收藏”，使卡片视觉清爽大方。

### 2.2 自动去背景与食物轮廓提取算法
1. 上传图片后，通过 `@imgly/background-removal` 获得去除背景后的透明 PNG 图片。
2. 将透明图片绘制在临时 Canvas 上，通过 `ctx.getImageData()` 提取像素。
3. 执行**边缘查找算法**：遍历每个像素的 alpha 值，定位不透明度大于 20 的像素边缘，提取轮廓点集。
4. 在渲染 Canvas 上绘制食物图片，并在不规则的轮廓路径上，使用 `ctx.setLineDash([8, 6])` 绘制白色轮廓虚线，输出轮廓抠图贴纸。

## 3. 技术方案与数据架构

### 3.1 核心技术选型
* **前端框架**：React + Vite 驱动。
* **离线抠图与轮廓算法**：`@imgly/background-removal` 配合自定义的 HTML5 Canvas 边缘像素检测提取路径。
* **本地存储**：IndexedDB 实现大容量图片贴纸的离线保存。

### 3.2 数据库 Schema (IndexedDB)
```typescript
interface FoodRecord {
  id: string;          // 唯一 UUID
  timestamp: number;   // 记录时间戳
  foodName: string;    // 食物名称
  mealType: 'breakfast' | 'lunch' | 'dinner' | 'tea' | 'night';
  rating: number;      // 1-5 评分
  isNewFood: boolean;  // 是否为新食物
  diningWith?: string; // 和谁吃
  location?: string;   // 在哪里吃
  isFavorited: boolean;// 是否收藏
  note: string;        // 备注日记
  imageBlob?: Blob;    // 抠图去背景后的透明 PNG 数据（包含合成白色轮廓虚线）
}
```
