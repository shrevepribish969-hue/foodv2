# “吃点好的” — 智能治愈系饮食记录应用设计文档

本项目是一个面向日常生活记录的智能饮食 Web 应用，旨在通过可爱的视觉风格和独特的“手绘圈出食物”手账交互，让日常饮食记录从单纯的“健康管理”转变为“可回忆、可分享的生活回忆录”。

## 1. 核心视觉与交互设计

### 1.1 视觉风格 (UI/UX)
* **配色调性**：采用柔和温润的粉彩马卡龙色调。
  * 背景色：暖奶油白（#FFFDF6）辅以可爱的网格线背景，模拟手账本的网格纸纹理。
  * 卡片及辅助色：粉色（#FFB6C1）、薄荷绿（#A8E6CF）、香蕉黄（#FFD3B6）。
  * 按钮及高亮：带有弹性的 bouncy 交互动画，大圆角（`border-radius: 24px`），软萌柔和的阴影。
* **字体**：使用圆润易读的圆体，数字和英文选用 `Fredoka` 或 `Quicksand` 字体。

### 1.2 页面结构 (多视图切换)
* **【食光轴】(首页)**
  * **今日餐卡**：固定展示在顶部，展示今天的用餐打卡进度。
  * **竖向时间轴**：轴线使用可爱虚线，轴上按时间倒序排列饮食记录卡片（最新的一餐在最上方）。
  * **时间线卡片内容**：
    * 左侧：用户手动画出虚线圈的食物照片（若是纯文字记录，显示系统预设的治愈风手绘食物图标）。
    * 右侧：食物名称、餐时标签（🌸 早餐、🌤️ 午餐、🍵 下午茶、🌙 夜宵等）、用餐时间、心情爱心打分、备注。
* **【全景墙】(月视图)**
  * 采用经典的日历网格（Calendar Grid）展示当月的每一天。
  * 每一个日期格内展示该天吃过的食物抠图/虚线贴纸缩略图。
  * 点击具体的日期格，弹出该天详细的饮食时间轴记录。
* **【回忆录】(月报模式)**
  * 生成一张可爱的长卡片，分析当月饮食的趣味数据：
    * 🍽️ 累计用餐次数（例如：“本月您享用了 82 顿饭”）
    * 🆕 尝鲜新食物（首次享用的食物计数，如：“尝试了 15 种新食物”）
    * 💖 最爱美食（打分最高且频次最高的食物名称）
    * 🔁 最常享用（词频统计最高的食物名称）
    * 🌙 深夜食光（晚上 21:00 至凌晨 4:00 间记录时间最晚的一餐）
    * 📊 平均每日用餐数（如：“平均每天享用 2.8 餐”）
  * 提供“导出为图片”按钮，将卡片合成为一张精美的手账长图以便分享。

## 2. 核心交互流程

### 2.1 记录新增弹窗
1. 点击主界面的悬浮 `➕` 按钮。
2. 上传照片（或选择纯文字记录）。
3. 填写食物名字（支持历史输入联想）、选择餐时类型、调整时间、评分（1-5颗心）、勾选“是否第一次吃”。

### 2.2 手绘圈出食物（Lasso 虚线画布）
当用户上传照片后，系统自动在照片上方覆盖一个 HTML5 Canvas 画布，引导用户手动在食物上“画个圈”：
* **画笔线条**：用户拖拽/涂抹生成的路径会自动平滑并绘制为一条**带有手绘抖动感的虚线**。
* **画笔颜色**：提供 4 种马卡龙笔触颜色（粉、黄、白、绿）供选择。
* **重画与保存**：如果不满意可一键清除重画，完成后点击“画好了”保存。

## 3. 技术路线与数据结构

### 3.1 核心技术栈
* **框架**：React + Vite (单页面 Web 应用)。
* **样式**：Vanilla CSS，精细控制治愈系手账的网格纹理、圆角及弹性交互动画。
* **数据存储**：由于包含大图资源，使用浏览器自带的 **IndexedDB** 存储，实现完全的本地离线存储，数据隐私安全，刷新不丢失。

### 3.2 数据结构 (Schema)
每一条饮食记录在 IndexedDB 中的存储结构如下：
```typescript
interface FoodRecord {
  id: string;          // 唯一标识符 (UUID)
  timestamp: number;   // 用餐时间戳
  foodName: string;    // 食物名称
  mealType: 'breakfast' | 'lunch' | 'dinner' | 'tea' | 'night'; // 餐时
  rating: number;      // 喜爱评分 (1-5)
  isNewFood: boolean;  // 是否为尝鲜新食物
  note: string;        // 备注日记
  imageBlob?: Blob;    // 原始上传图片的二进制数据
  lassoPoints?: { x: number; y: number }[]; // 手绘虚线的点集坐标（便于在响应式画布上自适应缩放绘制）
  lassoColor?: string; // 虚线笔触颜色
}
```

### 3.3 图片与Canvas渲染机制
在时间线和日历卡片渲染时：
1. 使用一个只读的 `<canvas>`，首先将食物原图渲染为背景。
2. 根据 `lassoPoints` 点集坐标与当前 Canvas 容器的比例，动态利用 `ctx.setLineDash([6, 6])` 重新绘制可爱的虚线描边，实现高清晰度、自适应大小的贴纸效果。
