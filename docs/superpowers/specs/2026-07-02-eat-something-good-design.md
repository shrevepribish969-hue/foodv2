# “吃点好的” — 智能治愈系饮食记录应用设计文档

本项目是一款面向日常生活记录的智能饮食 Web 应用，以“吃点好的”为核心主题。通过治愈系的手账日记风格，将日常饮食记录转化为“可回忆、可收藏的生活回忆录”。

---

## 1. 核心视觉与页面规划

### 1.1 页面视觉设计 (UI/UX)
* **配色调性**：采用柔和温润的粉彩马卡龙色调。
  * 背景色：暖奶油白（#FFFDF6）辅以可爱的网格线背景，模拟手账本的网格纸纹理。
  * 卡片及辅助色：樱花粉（#FFB6C1）、薄荷绿（#A8E6CF）、香蕉黄（#FFD3B6）。
  * 交互体验：大圆角卡片（`border-radius: 24px`），按钮和过渡带有弹性 bouncy 动画。
* **食物贴纸效果**：用户上传的照片通过 **AI 自动去除背景 (AI Background Removal)** 提取出真实的食物，并自动在其周围环绕一圈精致的白色虚线描边，形成高还原度的“手账实体食物贴纸”。

### 1.2 四大核心功能板块
1. **【今日饮食 - Today】(首页)**
   * **日记翻页概念**：首页即为“今天的一页”，每天就是一页手账，一年累积起来就是一本 365 页的饮食日记。
   * 支持通过左右翻页按钮或滑动在不同的日期页面之间切换。
   * **单页内容**：展示该天记录的所有饮食卡片（早餐、午餐、晚餐、下午茶、夜宵等）。
   * 每个卡片展示：餐时标签、抠图食物贴纸、食物名字、评分、备注、和谁吃、在哪里吃。
2. **【月视图】**
   * 以传统的日历网格展示整月。
   * 每个日期格中会平铺展示当天所有饮食记录的食物贴纸缩略图，形成一面绚丽的“当月食物贴纸墙”。
   * 点击日期直接跳转到该天的 **Today's Page**。
3. **【回忆录】(月报模式)**
   * 自动生成一份精美的长图卡片，展示当月的数据回忆：
     * **AI 温暖总结**：调用 AI（Gemini）根据用户当月的饮食历史、聚餐频次和心情，生成一段温暖治愈的月度总结（例如：“这个月你尝试了许多新的味道，也和重要的人度过了温暖的饭局...”）。
     * **趣味统计指标**：
       * 🍽️ 累计用餐次数
       * 🆕 尝鲜新食物（首次记录的食物数）
       * 🥤 饮品次数（统计名字中包含奶茶、咖啡、可乐、饮品等的记录）
       * 👥 朋友聚餐次数（统计有“和谁吃”的记录数）
       * 💖 最偏爱与最常吃（基于评分与词频统计）
       * 🌙 深夜食光（21:00 - 4:00 最晚用餐记录）
       * 📍 常去地点（“在哪里吃”的地点频次分析）
     * 提供一键“导出为图片”功能。
4. **【收藏推荐页】**
   * 展示所有被用户标记为“收藏”的饮食回忆卡片，形成个人专属的“美食推荐清单”和“生活高光时刻”。

## 2. 交互与功能流程

### 2.1 “吃点好的” 记录新增
用户点击悬浮的 `➕` 按钮，弹出记录抽屉/弹窗：
1. **照片上传与 AI 去背景**：
   * 拖拽或上传实物照片，前端调用纯本地 WebAssembly 抠图库（如 `@imgly/background-removal`）自动秒级剥离食物背景。
   * 剥离背景后，自动在食物轮廓边缘追加 2px 白色虚线。
   * *纯文字模式/未上传照片*：系统随机分配一个手绘卡通食物图标作为占位贴纸。
2. **详细信息输入**：
   * 食物名称（支持历史名称智能联想输入）。
   * 餐时类型（早餐/午餐/下午茶/晚餐/夜宵）。
   * 用餐时间（默认为当前时间，支持微调）。
   * 和谁吃（输入人名，用于统计聚餐）。
   * 在哪里吃（输入地点，用于统计常去地点）。
   * 是否第一次吃（尝鲜标记）。
   * 喜爱评分（1-5颗粉色爱心）。
   * 备注日记（心情与备忘）。
   * **收藏按钮**（点击可将此顿饭加入“收藏推荐页”）。

## 3. 技术方案与数据架构

### 3.1 核心技术选型
* **前端框架**：React + Vite 驱动。
* **样式设计**：Vanilla CSS 样式系统，用于搭建极具质感的手账本阴影、缝线和纸张质感。
* **离线抠图**：使用 `@imgly/background-removal` 实现本地离线去背景，辅以 Canvas 进行边缘路径描边。若库加载慢或失败，则自动切换到**精美心形/圆角裁剪贴纸**作为兜底。
* **本地存储**：由于需要缓存高清去背景 PNG 贴纸，采用 **IndexedDB** 保证完全的本地存储空间。
* **AI 总结服务**：在月报页面提供一个“AI 总结”按钮，用户可配置 Gemini API Key（安全存储在 `localStorage` 中）实现纯前端的分析总结。

### 3.2 数据库 Schema (IndexedDB)
```typescript
interface FoodRecord {
  id: string;          // 唯一 UUID
  timestamp: number;   // 记录时间戳
  foodName: string;    // 食物名称
  mealType: 'breakfast' | 'lunch' | 'dinner' | 'tea' | 'night';
  rating: number;      // 1-5 评分
  isNewFood: boolean;  // 是否为新食物
  diningWith?: string; // 和谁吃
  location?: string;   // 在哪里吃
  isFavorited: boolean;// 是否收藏
  note: string;        // 备注日记
  imageBlob?: Blob;    // 抠图去背景后的透明 PNG 数据
}
```
