# 手账贴纸双层白边与明丽橘黄色线条设计

本设计旨在根据用户的参考图，实现兼具可爱感与手账质感的“双重贴纸描边”效果，同时解决自动抠图边缘不够干净的问题：
- **白色扩边底**：在抠出来的食物边缘向外扩展一圈 8px 宽的纯白色底框。这不仅能使食物像实体剪纸贴纸一样精致，还能天然吞没和遮盖抠图边缘的毛刺与不干净的噪点。
- **橘黄色边缘线**：在白色边框的最外侧，描画一圈厚度为 3px 的明丽橘黄色连续实线。
- **背景透明化**：抠图成功时保持背景透明，使最终呈现为一张带白边和橘边的高品质食物贴纸。

## 方案设计

### 1. 双层描边渲染流程

修改 [RecordModal.tsx](file:///d:/A研二/food2.0纯记录/foodV2/src/components/RecordModal.tsx) 的 `renderCanvas` 函数。
在 `imgCutout` 存在时：
1. 不绘制底层原图 `imgOriginal`，不绘制压暗遮罩。
2. 绘制橘黄色剪影：创建 `orangeSilhouette` 离屏 Canvas，绘制 `imgCutout` 并使用 `'source-in'` 填充明丽的橘黄色（如 `#FF9800`）。
3. 绘制白色剪影：创建 `whiteSilhouette` 离屏 Canvas，绘制 `imgCutout` 并使用 `'source-in'` 填充纯白色（`#FFFFFF`）。
4. 在主 Canvas 上朝 16 个方向平移 `11px`（8px 白框 + 3px 橘线）绘制橘黄色剪影。16 个偏移方向能确保大平移量下的线条绝对连续且平滑。
5. 在主 Canvas 上朝 16 个方向平移 `8px` 绘制白色剪影，盖在橘黄色图层上方。
6. 最后在正中心（0px 偏移）绘制彩色食物 `imgCutout`。

抠图失败兜底时（`imgCutout` 不存在）：
- 显示原图并叠加径向压暗，并用橘黄色（`#FF9800`）绘制圆形虚线描边。

## 验证计划

- 上传一张食物图片，观察抠图完成后的预览图：
  - 是否有明显的白色贴边将食物包围。
  - 白色贴纸边缘是否环绕有一圈连续、平滑、醒目的橘黄色粗实线。
  - 抠图边缘是否比之前显得更干净、平整。
- 测试 WASM 抠图失败时的兜底（圆形虚线）描边，确保圆形虚线颜色为明丽的橘黄色。
