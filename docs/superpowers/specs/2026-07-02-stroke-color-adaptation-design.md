# 手账贴纸双层白边与明丽橘黄色描边设计 (紧凑自适应裁剪版)

本设计旨在根据用户的反馈，彻底解决食物抠图后“因为保留原图长宽比而产生大块透明留白，导致贴纸在卡片中显示过小”的痛点：
- **食物主体边界检测 (Bounding Box Detection)**：读取抠图后的图像，自动扫描像素。找出所有不透明度 `Alpha > 20` 的像素点的极值边界 `[minX, minY, maxX, maxY]`。这个最小包围盒能精准定位到食物与餐具本身，排除原照片中被抠除的所有周围桌面大面积空白。
- **去除空白边缘裁剪**：仅将包围盒内的食物主体区域裁剪提取。
- **适度最大化缩放**：仅针对裁剪后的主体计算 `scale = Math.min(260 / cropW, 260 / cropH)`，使得食物在 300x300 画布中以最大最饱满的比例居中呈现，同时保留 11px 描边的安全边缘。
- **边缘去噪硬化**：在裁剪后的主体像素中，将 `Alpha < 40` 的残留清零，`Alpha >= 40` 的像素强行硬化为 `255`，提供完美的描边底图。
- **白色扩边底**：在食物主体最外侧边缘外扩展一圈 8px 宽的纯白色底框。
- **橘黄色边缘线**：在白色边框的最外侧，描画一圈厚度为 3px 的明丽橘黄色（`#FF9800`）连续实线。
- **背景透明化**：抠图成功时保持背景透明。

## 方案设计

### 1. 双层描边渲染流程

修改 [RecordModal.tsx](file:///d:/A研二/food2.0纯记录/foodV2/src/components/RecordModal.tsx) 的 `renderCanvas` 函数。
在 `imgCutout` 存在时：
1. 不绘制底层原图 `imgOriginal`，不绘制压暗遮罩。
2. **包围盒计算**：用 `boxCanvas` 绘制 `imgCutout`。通过像素遍历找到不透明像素的极值边界，并添加 4px 的安全 Padding。得到裁剪尺寸 `cropW = maxX - minX + 1`，`cropH = maxY - minY + 1`。
3. **裁剪与去噪**：将包围盒区域 `[minX, minY, cropW, cropH]` 从 `boxCanvas` 裁剪并绘制至 `cleanCanvas`，利用像素过滤将 `Alpha < 40` 清零，`Alpha >= 40` 置为 `255`。
4. **自适应缩放比例**：以 `cropW` 和 `cropH` 为基准计算 `scale = Math.min(260 / cropW, 260 / cropH)`，得到缩放尺寸 `w`, `h` 及居中起始坐标 `x`, `y`。
5. **绘制橘黄色与白色剪影**：分别创建 `orangeSilhouette` 与 `whiteSilhouette` 离屏 Canvas，绘制已居中缩放定位的 `cleanCanvas` 图像，利用 `'source-in'` 分别填充 `#FF9800` 和 `#FFFFFF`。
6. 在主 Canvas 上朝 16 个方向平移 `11px`（橘色线外边缘）和 `8px`（白色外边缘）绘制剪影底色。
7. 正中心裁剪绘制彩色食物本体。

抠图失败兜底时（`imgCutout` 不存在）：
- 显示原图并叠加径向压暗，并用橘黄色（`#FF9800`）绘制圆形虚线描边。

## 验证计划

- 上传原照片（食物仅占中间一部分，周围有宽阔背景桌面）进行测试：
  - 验证生成的卡片中，去背景后的食物主体是否能够自动铺满画布，完全消除了四周大片多余的透明留白。
  - 描边线是否完好围着最大化食物的白色底框画圈，没有超出画布截断。
