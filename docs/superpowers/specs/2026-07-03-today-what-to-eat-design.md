# 收藏页【今天吃什么】贴纸袋撕开推荐设计规格书

本设计旨在将收藏页面（FavoritesPage）的“今天吃什么”卡片功能重构为手账纸袋撕开推荐交互。用户通过向右滑动拉锁，横向撕开美味贴纸袋，令里面封存的 4 张推荐美味贴纸向上滑出。

## 需求背景与功能设计

1. **入口卡片**：
   - 位置：[FavoritesPage.tsx](file:///d:/A研二/food2.0纯记录/foodV2/src/components/FavoritesPage.tsx) 搜索框正下方。
   - 样式：采用无印良品风格的暖黄纸袋卡片，左侧展示标题“今天吃什么？”、副标题“不知道吃什么？拆个美味贴纸包吧～”和“开始拆包”按钮；右侧展示一个手绘风格的密封贴纸袋 SVG。
2. **推荐数据与算法**：
   - **时间范围**：最近 30 天的饮食记录。
   - **过滤条件**：
     - 排除评分低于 3★ 的记录。
     - 排除最近 5 天内吃过的食物（避免重复推荐）。
   - **自适应多重兜底**：
     - 如果符合上述条件的食物数量不足，自动放宽排除最近 5 天的条件。
     - 如果最近 30 天没有符合条件的记录，自动获取历史所有评分 >= 3★ 的食物。
     - 如果数据库完全为空，自动加载 4 款精心设计的静态内置美食贴纸（重庆小面、奶油培根意面、麻辣火锅、提拉米苏），确保新用户可无障碍操作。
   - **好久没吃算法**：如果抽中的美味距离上次吃的时间在 20 天到 30 天之间，展示特殊的 **“💡 好久没吃”** 标识。
   - **样式一致性**：对于无抠图贴纸（无 `imageBlob`）的用餐，使用统一的极简 `Utensils` 刀叉线条图标（淡卡其色）作为 fallback。
3. **“撕开拉条”物理交互流程**：
   - 弹窗正中展示一个竖向的精致纸质贴纸袋（SVG 渲染），纸袋上方有一条**横向的撕裂拉锁虚线**。
   - 虚线左侧是一个带有“👉”箭头的滑块，提示“向右滑动以撕开袋口”。
   - 用户鼠标/手指**按住滑块向右拖动**：
     - 横向的撕裂带利用 `clipPath` 从左向右逐渐缩短并消失，模拟撕开效果。
     - 滑动满 100% 后，撕裂带完全消失，纸袋顶部开口露出。
   - **贴纸向上滑出**：
     - 随后，纸袋的上半盖消失，4 张符合筛选条件的美味贴纸以交错延迟动效（Staggered Animation）**从纸袋开口处向上滑出**，悬浮于上方。
   - **选定退出**：
     - 4 张贴纸并排或矩阵列出。用户点击其中任意一张贴纸，页面立刻关闭并返回，没有多余提示。没有“再来一次”或“换一个”按钮。

## 技术实现方案

### 1. 样式定义 ([index.css](file:///d:/A研二/food2.0纯记录/foodV2/src/index.css))

修改并追加贴纸包撕开、贴纸滑出的关键帧动画：

```css
/* 贴纸从纸袋开口向上滑出 */
@keyframes sticker-slide-up {
  0% {
    transform: translateY(60px) scale(0.85);
    opacity: 0;
  }
  100% {
    transform: translateY(0) scale(1);
    opacity: 1;
  }
}

.sticker-slide-active {
  animation: sticker-slide-up 0.5s cubic-bezier(0.175, 0.885, 0.32, 1.275) both;
}

/* 纸袋轻微呼吸/待撕裂动效 */
.bag-pulse {
  animation: bag-breathing 2s ease-in-out infinite;
}

@keyframes bag-breathing {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.02); }
}
```

### 2. 状态机逻辑定义

在贴纸袋弹窗中，通过以下状态控制展示：
- `bagState`: `'ready' | 'tearing' | 'opened'`
- `dragProgress`: `number` (0 到 100，表示向右拖动百分比)
- `recommendedStickers`: `FoodRecord[]` (一次性生成的 4 个推荐美味贴纸)
