# 子项目 B：抠图体验与样式升级 (Cutout UX & Styling)

## 项目背景
当前应用在处理图片抠图时，耗时较长（约 30 秒）且只显示滚动的文字进度，导致等待体验不佳。此外，抠图后的贴纸边缘带有橘色描边，与整体的极简审美不太契合，且在浅色背景下缺乏立体感。本项目旨在优化这部分的视觉体验和加载反馈。

## 核心设计

### 1. 抠图进度条可视化
- **交互与视觉**：在 `RecordModal` 中，当触发图片抠图算法时，将原本居中的“加载圆环+文字”替换为一条**水平进度条**。
- **进度计算**：利用 `@imgly/background-removal` 提供的 `progress` 回调中的 `current` 和 `total` 参数计算百分比，实时更新进度条的宽度。
- **UI 细节**：进度条采用圆角设计，主色调使用品牌绿（`var(--color-green)`），并在进度条上方显示简练的提示文本（如“正在提取食物贴纸... 45%”），下方保留耗时显示。

### 2. 贴纸样式优化：实体贴纸阴影效果
- **去除橘色边缘**：在 `RecordModal.tsx` 的 `renderCanvas` 方法中，移除原有的 16 方向橘黄色偏移绘制逻辑。
- **保留白色底边**：继续保留 8px 的白色偏移描边，确保贴纸具有实体的白边效果。
- **烘焙阴影**：在 Canvas 最终合成时，利用 Canvas API（`ctx.shadowColor`, `ctx.shadowBlur`, `ctx.shadowOffsetY` 等）为贴纸烘焙一层柔和的黑色半透明阴影（例如 `rgba(0, 0, 0, 0.15)`），使其看起来像一张真实的实体贴纸。这样阴影会被直接保存在 PNG 图片中，在任何页面展示都能保持立体感。

### 3. 贴纸预览区增加底色
- **视觉优化**：将 `RecordModal` 中用于展示贴纸预览的容器（目前为 `#FAF6EE`）改为一种对比度稍高一些的淡淡底色（例如带有轻微纹理或稍深一点的浅咖/米灰色 `#EFE9DF`），以确保白边和阴影能够更加清晰地凸显出来，让贴纸的立体感更明显。

## 开发范围
- **修改文件**：
  - `src/components/RecordModal.tsx`:
    - 更新 `progress` 状态的管理，新增 `progressPercent` 状态。

    - 重写 `renderCanvas` 的绘制逻辑，去掉 `orangeSilhouette`，加入 `shadow` 设置。
    - 调整预览框容器的 `background` 颜色。
