# 子项目 D：月历与回忆录数据增强 (Data & MonthView Enhancement)

## 项目背景
当前月历默认展示“喜爱度/最新”的贴纸，无法自定义；同时回忆录页面缺乏对于“花销”的可视化展示。子项目 D 旨在补全这两个维度的体验，提升应用的数据回顾能力和个性化。

## 核心设计

### 1. 月历长按更换贴纸
- **长按交互**：在 `MonthView` 的日历网格中，为每一天的卡片添加 `onPointerDown`/`Up` 事件。当长按某一天（比如 500ms）时，会弹出一个极简风格的贴纸选择弹窗。
- **贴纸选择弹窗 (`DayStickerSelectModal`)**：
  - 以网格形式展示这一天所有记录的贴纸图片（如果没有图片则显示文字或 Emoji）。
  - 用户点击其中一张后，将选择偏好保存到 `localStorage` 中（例如 `food_cover_prefs: { "2026-07-09": "record-id" }`）。
- **默认逻辑兼容**：`MonthView` 在渲染每一天时，优先读取偏好设置中的 `recordId`，如果该 `recordId` 存在于当天的记录中，则作为该天的展示贴纸；若不存在偏好或匹配失败，则降级使用原本的默认规则（评分降序+时间降序）。

### 2. 回忆录花销折线图
- **UI 布局**：在 `ReportPage`（回忆录）页面的“AI 月度回忆”模块下方，新增一个“花销追踪”模块。
- **时间维度切换**：模块顶部包含一组圆角分段选择器（Segmented Control），包含三个选项：“日”、“周”、“月”。
- **数据计算**：
  - **日 (Daily)**：展示最近 7 天的每日花销总和（基于 `price` 字段）。
  - **周 (Weekly)**：展示最近 4 周的每周花销总和。
  - **月 (Monthly)**：展示最近 6 个月的每月花销总和。
- **极简 SVG 折线图**：
  - 为了完美契合手账风格，不引入笨重的图表库，而是手写一个极简的自适应 `<svg>` 折线图。
  - 绘制主线（原木绿 `var(--color-green)`）、数据点（带白色描边圆点），并在 X 轴下方标注日期/周期文字。
  - 在数据点上方直接显示金额数值（带 `¥` 符号），保持界面克制无多余轴线。
