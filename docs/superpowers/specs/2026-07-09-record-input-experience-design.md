# 子项目 A：记录录入体验优化 (Record Input Experience)

## 项目背景
为了提升“吃点好的”APP的日常使用体验，针对卡片录入流程（录入弹窗及快速记录）进行了一系列优化。主要目标是让录入更快捷、信息更完善，同时增加高度定制化的分类标签。

## 核心设计

### 1. 闪念笔记（长按 + 号）
- **交互方式**：在 `TodayPage.tsx` 页面，用户长按右下角的加号（`+`）超过 500ms 时，触发系统震动（若设备支持）并弹出一个全新的“闪念笔记”弹窗。
- **界面设计**：极简风格，不再展示复杂的录入项。只包含：
  - 一个大的文本输入框，提示语为“此刻的想法...”。
  - 一个“保存”按钮。
- **数据结构**：保存时生成一条默认的 `FoodRecord`。其中 `foodName` 默认填充为“闪念笔记”，`note` 为输入的纯文本内容，无图片，`mealType` 为默认值或新增特定类型。

### 2. 相机与相册分离
- **交互方式**：在 `RecordModal.tsx` 中优化图片上传区域的操作。
- **默认拍照**：主占位区（大的 Camera 图标和提示区域）被点击时，默认唤起系统**相机**（通过 `<input type="file" accept="image/*" capture="environment">` 实现）。
- **相册入口**：在占位区的右下角添加一个次级图标按钮（使用 `lucide-react` 的 `Image` 或 `FolderSync` 图标，保持风格一致），点击该图标则唤起系统相册。

### 3. 增加价格字段
- **数据库变动**：修改 `db.ts` 中的 `FoodRecord` 接口，增加选填的 `price?: number` 字段。
- **界面设计**：在 `RecordModal.tsx` 的手账式表单中，紧贴“在哪里吃”的下方新增“价格”输入行。左侧文本为“价格”，右侧显示 `¥` 符号并跟随纯数字输入框。

### 4. 自定义分类标签
- **数据结构变动**：原本的 `mealType` 将转为支持自定义字符串。初始的默认标签在进入应用时初始化在 `localStorage` 中（默认值：`['早餐', '午餐', '晚餐', '饮品']`）。
- **界面与展示**：
  - 默认的四个标签（早餐、午餐、晚餐、饮品）在渲染时会继续匹配显示原有的对应小图标（Sun, Moon, Coffee 等）。
  - 用户自行添加的标签仅显示纯文字，保持极简视觉。
- **编辑交互**：
  - 在“餐时”标题旁边增加一个编辑图标（使用 `lucide-react` 的 `Pencil` 保持样式一致）。
  - 点击进入编辑模式后，每个标签末尾出现删除图标（`lucide-react` 的 `X` ），并且在列表末尾出现一个新增按钮（`lucide-react` 的 `Plus`）。
  - 删除时提示并移除标签，新增时弹出浏览器默认 `prompt` 或极简小弹窗让用户输入新标签名称，随后同步到 `localStorage` 中。

## 开发范围
- **修改文件**：
  - `src/db.ts`: 更新 `FoodRecord` 类型，支持 `price`。
  - `src/components/TodayPage.tsx`: 添加 `+` 按钮的长按交互，引入 `QuickNoteModal`。
  - `src/components/RecordModal.tsx`: 修改图片上传控件，添加相册入口，增加价格输入行，重构餐时标签为动态列表，并实现标签编辑态。
- **新增文件**：
  - `src/components/QuickNoteModal.tsx`: 用于闪念笔记极简文字录入的独立组件。
